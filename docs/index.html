Uma vaga no meio.

API REST para gestão de vagas: empresas publicam, candidatos se inscrevem — cada lado com seu próprio login e seu próprio token JWT.

Ver repositório → Ver rotas da API

Este projeto é um backend puro (Java/Spring Boot) — não existe tela para navegar aqui. Esta página documenta a API; para testar de verdade, rode o projeto localmente com Docker (passo a passo abaixo) e abra o Swagger.

Java 17 Spring Boot 3.5 PostgreSQL Spring Security · JWT Docker Compose Prometheus + Grafana
Arquitetura

Dois módulos, dois segredos de token

Um token de candidato não abre rota de empresa, e vice-versa — cada perfil tem seu próprio filtro de segurança e sua própria chave JWT.

Empresa

/company

POST /company/ Cadastra uma empresa
POST /company/auth Login da empresa
POST /company/job/ Publica uma vaga 🔒 auth
Candidato

/candidate

POST /candidate/ Cadastra um candidato
POST /candidate/auth Login do candidato
GET /candidate/ Perfil do candidato 🔒 auth
GET /candidate/job Lista vagas (com filtro) 🔒 auth
POST /candidate/job/apply Candidata-se a uma vaga 🔒 auth
Rodando local

Do clone ao primeiro token

Precisa de Docker e Docker Compose instalados.

  1. Clone e entre na pasta 
    git clone https://github.com/GlawberSantos/gestao-vagas.git
cd gestao-vagas
  2. Crie o arquivo .env

    Com DATABASE_URL, DATABASE_USERNAME, DATABASE_PASSWORD, JWT_SECRET_COMPANY e JWT_SECRET_CANDIDATE — gere os dois segredos com openssl rand -base64 32.

  3. Suba os containers 
    docker-compose up -d
  4. Abra o Swagger

    Com a API no ar em localhost:8080, a documentação interativa fica em /swagger-ui/index.html.
